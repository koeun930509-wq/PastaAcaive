# 파스타 레시피 아카이빙 서비스 PRD (Product Requirements Document)

- **문서 버전**: v1.0
- **작성일**: 2026-07-04
- **작성 방식**: AI 인터뷰 기반 아이디어 구체화

---

## 1. 개요 (Overview)

### 1.1 서비스 소개
SNS(유튜브·인스타그램 등)에 흩어져 있는 파스타 레시피 영상을 **요약 정리**하고, 실제로 만들어본 **후기**를 기록·공유하는 반응형 웹 서비스. 운영자(Master)가 직접 검증한 레시피 후기와, 사용자(User)들이 자유롭게 남기는 후기가 함께 쌓이는 구조.

### 1.2 해결하고자 하는 문제
- 파스타 레시피 영상은 넘쳐나지만, **실제로 만들어봤을 때 맛있는지에 대한 신뢰할 만한 후기가 부족**함
- 레시피 정보(재료·순서)와 실사용 후기가 분리되어 있어, 믿고 따라 할 레시피를 찾기 어려움

### 1.3 핵심 차별점
- 종합 레시피 플랫폼(만개의레시피 등)이나 SNS 자체 검색과 달리, **"파스타"라는 하나의 니치에만 집중**
- 잡다하지 않고 한 주제에 몰입된 정보 큐레이션 제공
- Master가 직접 만들어보고 남기는 신뢰 기반 콘텐츠 + User들의 실사용 후기가 이중으로 쌓이는 구조

- 레시피 영상을 끝까지 보지 않고도 핵심만 빠르게 파악하고 싶은 사람들

### 3.2 메인 페이지 (Main Page)

| Master 추천 레시피 | 갤러리형 / 리스트형 토글 가능. 카드에 **사진 + 파스타 이름** 표시. **모바일에서는 3번째 카드가 절반만 보이는 스와이프형 가로 스크롤**(화살표 버튼으로도 이동 가능), **PC에서는 5열 그리드**로 전환 |
| User 레시피 | Master와 동일한 형식 (사진 + 파스타 이름, 모바일 스와이프 / PC 그리드) |
| Footer | 서비스 정보 |

> 카테고리(토마토·크림·오일·로제·바질·기타) 바로가기는 메인에는 넣지 않고, Master/User Review 페이지 진입 시 **탭 메뉴**로 배치.

### 3.3 Master Review 페이지
- 진입 시 **탭 메뉴**: 토마토 / 크림 / 오일 / 로제 / 바질 / 기타
- 후기 목록은 **썸네일(직접 촬영한 결과물 사진) 갤러리**로 표시
- 썸네일 클릭 → 상세 페이지 진입

**Master 상세 페이지 구조 (순서대로)**
0. 레시피 이름(제목, 크게 표시) + 파스타 종류 카테고리 뱃지 (예: #토마토)
1. 결과물 사진 (Master가 직접 요리한 완성 사진)
2. 원본 영상 링크
3. 난이도 / 조리시간 태그
4. 재료 (전체 목록)
5. 레시피 (1, 2, 3, 4… 번호 순서로 요약)
6. 한 줄 후기 + 별점 (작성자 본인의 의견)
7. **한줄의견(댓글)** — 다른 방문자들이 **닉네임만 입력**하면 로그인 없이 자유롭게 짧은 의견을 남길 수 있는 영역. 작성자의 후기와 방문자들의 의견이 함께 쌓여 레시피 하나를 두고 소통하는 구조. **관리자(Master)가 로그인하면 악성 댓글을 삭제**할 수 있음

**별점 시스템 (커스텀 이모지 게이지)**
- 0~5점, 1점 단위
- 왼쪽부터 점수만큼 🍽️(싹싹 비움 = 존맛)로 채워지고, 나머지는 🍝(남음 = 노맛)로 채워지는 5칸 게이지
  - 예) 5점: 🍽️🍽️🍽️🍽️🍽️
  - 예) 3점: 🍽️🍽️🍽️🍝🍝
  - 예) 0점: 🍝🍝🍝🍝🍝

### 3.4 User Review 페이지
- Master Review와 동일한 탭 메뉴 구조(카테고리별)
- 후기 목록은 썸네일 갤러리로 표시 → 클릭 시 상세 페이지 진입

**User 상세 페이지 구조 (Master와 유사, 입력 방식만 다름)**

| 항목 | 규칙 |
|---|---|
| 레시피 이름 / 카테고리 | 제목을 크게 표시 + 파스타 종류 카테고리 뱃지(예: #크림) 노출 |
| 썸네일 | User가 직접 찍은 사진을 올리면 그 사진 사용 / 사진이 없으면 **유튜브 원본 링크에서 썸네일 자동 추출** (※ MVP는 유튜브만 지원, 인스타그램은 추후 확장) |
| 원본 영상 링크 | 선택 입력 (필수 아님, 유튜브 링크만 지원) |
| 재료 / 레시피 요약 | **유튜브 링크를 입력하면 AI가 자동으로 재료·조리순서 요약** 생성 / **링크가 없거나 유튜브가 아닌 경우 사용자가 직접 입력** 가능 |
| 한 줄 후기 + 별점 | Master와 동일하게 🍽️/🍝 게이지 별점(0~5점) 사용 (작성자 본인의 의견) |
| 한줄의견(댓글) | Master 상세와 동일하게, 닉네임만 입력하면 로그인 없이 짧은 의견을 남길 수 있음. 관리자(Master) 로그인 시 악성 댓글 삭제 가능 |

> 즉, User Review는 "사진 有無"와 "링크 有無" 두 가지 조건에 따라 입력 흐름이 유동적으로 분기되는 구조.
> - 사진 O + 링크 O: 사진을 썸네일로 사용, 재료/레시피는 AI 자동 요약
> - 사진 X + 링크 O: 링크 썸네일 자동 추출, 재료/레시피는 AI 자동 요약
> - 링크 X (사진 유무 무관): 재료/레시피는 사용자가 직접 입력

### 3.5 글쓰기 폼 (작성 페이지)
Master/User가 레시피 후기를 등록할 때 사용하는 작성 화면. 상세(view) 페이지와 동일한 순서·레이아웃을 기반으로 구성하되, 텍스트 대신 입력 요소로 채워짐.

| 입력 요소 | 동작 |
|---|---|
| 영상 링크 input | 유튜브 링크를 입력하면 **AI가 자동으로 재료·조리순서 요약** 생성 |
| 사진 file input | 완성 사진을 업로드하면 **자동으로 4:3 비율로 중앙 크롭**하여 썸네일로 사용 (어떤 비율로 업로드해도 4:3으로 통일) |
| 썸네일 fallback | 사진을 업로드하지 않으면 입력된 유튜브 링크에서 **썸네일 자동 추출** |
| 난이도 / 조리시간 태그 | 선택 입력 |
| 재료 / 레시피 | AI 자동 요약 결과가 채워짐 (링크 없을 시 사용자 직접 입력 가능) |
| 후기 textarea | 자유 텍스트 후기 작성 |
| 별점 | 🍽️/🍝 게이지 방식으로 0~5점 선택 |

> 별도의 "작성 페이지"로 존재하되, 시각적 순서(썸네일→링크→난이도/시간→재료→레시피→후기+별점)는 view 페이지와 동일하게 구성해 사용자가 "무엇을 입력하면 어떻게 보일지" 직관적으로 예측 가능하도록 함.

### 3.6 Pasta Recipe 요약 (핵심 기능)
- **최우선 MVP 기능**
- 사용자가 **유튜브** 레시피 영상 링크를 입력하면 **AI가 자동으로 영상을 분석**하여:
  - **재료 전체 목록** 정리
  - **1, 2, 3, 4… 번호 형식의 조리순서 요약** 생성
- 완전 자동화 방식 (링크만 입력하면 AI가 요약까지 완료)
- **요약 결과 복사 버튼** 제공 — 재료+레시피 텍스트를 클립보드로 한 번에 복사 가능 (다른 메모앱 등에 붙여넣기 용이)
- ※ MVP 단계에서는 **유튜브만 지원**, 인스타그램은 API·정책 제약으로 추후 확장 검토 (백로그 참고)

## 4. 핵심 기능 목록 (Feature List)

### MVP (1차 개발 범위)
1. **AI 자동 레시피 요약** — 영상 링크 → 재료+조리순서 자동 생성 (★최우선)
2. 메인 페이지 (Hero, 오늘의 파스타 레시피, Master/User 갤러리 미리보기)
3. Master Review 등록 및 상세 페이지 (사진, 링크, 난이도/시간, 재료, 레시피, 후기+별점)
4. User Review 등록 (링크 입력 → 썸네일 자동 추출)
5. 카테고리 탭 필터링 (토마토/크림/오일/로제/바질/기타)
6. **한줄의견(댓글) 기능** — 닉네임 입력만으로 방문자가 의견 작성 가능, 관리자(Master) 로그인 시 악성 댓글 삭제

### 2차 개발 (Backlog)
- 구글 소셜 로그인
- 스크랩(찜) 기능 — 찜한 레시피 모아보기 마이페이지
- 인기 랭킹 섹션 (별점 데이터 누적 후 "오늘의 파스타 레시피" → "인기 파스타"로 확장)
- 인스타그램 링크 지원 (썸네일 추출 + AI 요약 확장)

## 6. 기술적 고려사항 (Technical Considerations)

- **MVP는 유튜브만 지원**
  - 유튜브 공식 oEmbed API로 썸네일 추출 가능
  - 유튜브 자막(캡션) API 또는 오디오 STT로 레시피 요약 파이프라인 구축
- **인스타그램은 2차 이후 확장 검토 (백로그)**
  - Meta 앱 심사·토큰 발급 필요, 정책 변경 리스크 있어 MVP 범위에서 제외
  - 비공식 스크래핑 방식은 이용약관 위반 소지 있어 지양
- **AI 영상 분석 → 자동 레시피 요약**이 핵심 기술 과제
  - 결과 포맷: 재료 리스트 + 번호형 조리순서
- 로그인은 **구글 소셜 로그인** 방식 예정 (일반 사용자 대상, 2차 개발)
- **관리자(Master) 인증**은 별도의 경량 로그인으로 MVP에서부터 필요 — 댓글 삭제 등 콘텐츠 관리 권한용 (일반 사용자 구글 로그인과는 별개)
- 스크랩 기능은 로그인 이후 사용자별 데이터로 관리 (2차 개발)
- 개발은 AI(Claude)와 함께 진행

| 7 | User Review 작성 | `/user/new` | 일반 사용자가 후기 등록 (사진 선택, 링크 선택) | §4-1, §4-4 | 사진 X + 링크 O 시 자동 썸네일추출 안내 문구, 링크 X 시 재료/레시피 직접 입력 폼으로 전환되는 조건부 UI 필요 |
| 8 | Pasta Recipe 요약 (조회 도구) | `/summary` | nav 3번째 메뉴. 후기 등록과 무관하게 링크만 넣으면 AI 요약 결과를 바로 확인 | §4-1 | 링크 input 1개 + 결과 출력 영역(재료/레시피), "이 요약으로 후기 작성하기" CTA → 작성 폼으로 데이터 전달 |
| 9 | 로그인 (2차) | `/login` | 구글 소셜 로그인 | §4-Backlog | 스크랩 등 로그인 필요 기능 접근 시 리다이렉트 진입점 |
| 10 | 마이페이지 - 찜 목록 (2차) | `/mypage/scraps` | 사용자가 찜한 레시피 모아보기 | §4-Backlog | 로그인 필요, Master/User 구분 없이 통합 리스트 |

### 8.2 사이트맵

```
메인 (/)
│
├─ Master Review (/master)
│   ├─ [카테고리 탭: 토마토ㆍ크림ㆍ오일ㆍ로제ㆍ바질ㆍ기타] (같은 경로 내 필터)
│   ├─ Master Review 상세 (/master/[id])
│   └─ Master Review 작성 (/master/new)
│
├─ User Review (/user)
│   ├─ [카테고리 탭: 토마토ㆍ크림ㆍ오일ㆍ로제ㆍ바질ㆍ기타] (같은 경로 내 필터)
│   ├─ User Review 상세 (/user/[id])
│   └─ User Review 작성 (/user/new)
│       └─ (링크 X 시) 재료/레시피 수동입력 모드로 전환
│
├─ Pasta Recipe 요약 (/summary)
│   └─ "요약으로 후기 작성하기" → /master/new 또는 /user/new 로 데이터 전달
│
└─ (2차 개발 - Backlog)
    ├─ 로그인 (/login)
    └─ 마이페이지 - 찜 목록 (/mypage/scraps)
        └─ 로그인 필요 (미로그인 시 /login 리다이렉트)
```
